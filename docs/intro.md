# 课程介绍

欢迎来到 **HashiCorp Vault 交互式教程**！

## 机密蔓生：一个被低估的危机

当我们讨论信息安全时，不论是身份认证、权限控制、数据加密，一个绕不过去的核心问题就是对**机密（Secret）**的管理。所谓机密，就是用以代表某个特定身份来执行认证与授权管理的信息——数据库的账号密码是机密，调用 API 的令牌（Token）是机密，用来认证服务器与客户端身份的证书是机密，加解密关键数据的密钥也是机密。

在传统的研发场景中，我们经常能看到这样的问题：

- 生产环境的数据库用户名密码以明文形式编码在配置文件里
- 某个应用调用云服务接口所用的令牌被硬编码在代码中，随后不小心被上传到了公共 GitHub 仓库
- 掌握机密的员工离职后可能泄密，或是恶意报复
- 执行密码轮换策略极其痛苦，导致机密长期不换
- 需要将机密安全加密存储，但又不想将密钥暴露给应用程序

这种现象被称为[**机密蔓生（Secret Sprawl）**](https://www.hashicorp.com/en/resources/what-is-secret-sprawl-why-is-it-harmful)——系统机密信息被零散地以不同形式保存在许多彼此不相关的地方，我们既不知道哪些系统保存了哪些机密，也不知道哪些机密被哪些人获取过。即使发现某个机密信息流失在外，我们也无法回溯是何人申请使用、何人泄漏，更无法确信立即吊销该机密不会引发生产故障。

机密蔓生引发了一连串的严重问题，导致我们既不能事前防备机密泄漏，也无法事后快速响应进行损害管制。

## 动态防御：Vault 的核心思想

传统的安全体系往往被设计成静态防线——把防御设在系统的边界上，如同马奇诺防线，认为只要把外部流量隔离在外就万事大吉。遗憾的是，这种成本高昂的静态防线通常起不到什么作用：内部员工本身可能是恶意攻击者；一旦某个机密泄漏，因为难以更换，攻击者可以持续利用；层层审批虽然无法彻底消除风险，却成功给正常开发维护工作带来了巨大障碍。

HashiCorp Vault 针对安全防护提出了截然不同的思路——**动态防御**：

> 这一秒使用的密码，可能下一秒就变了。一切的机密都绑定了一个尽可能短的生命周期，即使攻击者攻入了系统纵深，面临的仍然是一个高度设防、高度动态的流动环境。旧时代的安全防护尽量阻止变更，而云原生时代高频率的变更本身就是安全防护。

Vault 的核心理念是：集中管理机密，建立机密的**单一事实来源（Single Source of Truth）**，对谁有权使用什么样的机密、谁正在使用这些机密、某个机密是谁申请的进行集中管理，并通过**动态机密**（按需生成、用完即废弃）从根本上解决机密泄漏的问题。

## 一个具体的例子

考虑一个常见场景：一个应用需要访问 AWS。传统做法是把 Access Key 和 Secret Key 写死在配置文件里，或者通过环境变量注入——这些 Key 往往几个月甚至几年都不更换。

使用 Vault 的 AWS Secrets Engine 后，工作方式变为：管理员将高权限的 AWS 根凭证保存在 Vault 里，并配置好角色和权限策略；当应用需要操作 AWS 时，临时向 Vault 申请一组专属的临时凭证：

```bash
$ vault read aws/creds/my-role
Key                Value
---                -----
lease_id           aws/creds/my-role/abc123
lease_duration     2m          # 2 分钟后自动过期
access_key         AKIAIOSFODNN7EXAMPLE
secret_key         wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

这组 Key 是 Vault 动态生成的临时凭证，有效期可以短至几分钟。用完后 Vault 会自动调用 AWS API 将其删除，无需任何人工干预。即使这组 Key 不慎泄漏，攻击者也来不及利用它——因为它已经过期了。

这就是 Vault 的动态机密（Dynamic Secrets）能力，也是它区别于传统密钥管理工具最核心的特性。

## 本教程涵盖的内容

本教程将带你从零开始，系统掌握 HashiCorp Vault 的核心能力：

- **Vault 基础**：Dev 模式、封印与解封、基本操作
- **机密引擎**：KV 静态存储、AWS/数据库动态机密、Transit 加密即服务
- **认证方法**：Token、AppRole、GitHub、用户名密码
- **访问策略**：编写 Policy，实现最小权限原则
- **实战案例**：生成动态数据库凭证、一次性 SSH 密码、生产环境安全加固
