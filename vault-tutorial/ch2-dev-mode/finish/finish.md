## Dev 模式的终极判断标准

**以下三条，任意一条为真，就不应使用 Dev 模式：**

1. 你的数据在服务器重启后需要保留
2. 不止一个人访问这个 Vault 实例
3. 这个 Vault 上有任何真实的、需要保密的数据

## 建立正确的 Vault 安全心智模型

Dev 模式是 HashiCorp 故意提供的一个"学习工具"，它通过牺牲所有安全保障来换取零摩擦的上手体验。理解了 Dev 模式的每一个设计妥协，你也就理解了真正的生产 Vault 为什么必须：

- 使用持久化存储（Raft）而非内存
- 强制开启 TLS（`tls_disable = false`）
- 随机生成 Root Token 并在初始化后立即吊销
- 通过 Shamir 或 Auto-Unseal 多方保障解封密钥的安全
- 开启审计设备记录所有操作
