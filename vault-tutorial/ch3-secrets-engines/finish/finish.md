# 恭喜完成实验！🎉

你已经在一个真实的 Vault 服务器上把机密引擎的**路由 + 生命周期 + Barrier View 隔离**这套核心模型完整跑了一遍。

## 本实验的核心收获

| 阶段 | 你亲手验证的事实 |
| :--- | :--- |
| **多实例挂载** | 同一种 KV 类型可以挂在多个不同路径下，每个实例拥有独立的 Accessor / UUID |
| **大小写敏感** | `case-test/` 与 `Case-Test/` 是两个完全独立的挂载点，数据互不可见 |
| **前缀冲突** | 挂载点之间不能互为前缀，Router 永远只匹配唯一前缀 |
| **`tune`** | 在线修改默认/最大 Lease TTL，数据零影响、可反复执行 |
| **`disable` 是销毁式卸载** | UUID 子树被整体清空，重新 enable 同名路径得到的是新 UUID 的空引擎 |
| **Barrier View 隔离** | 一个引擎被 disable 不会影响其他引擎；同名 key 在不同实例下互不相干 |

## 关键概念图

```
┌────────────────────────────────────────────────────────────┐
│  vault secrets enable -path=X T                            │
│    ↓                                                        │
│  生成新 UUID，开辟一块 chroot 子树                           │
│  在路由表中注册：X/  →  Accessor → UUID                     │
└────────────────────────────────────────────────────────────┘
                              │
                              │
┌────────────────────────────────────────────────────────────┐
│  vault secrets tune X/                                     │
│    ↓                                                        │
│  改运行参数（TTL 等），数据零影响                            │
└────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────┐
│  vault secrets move X/  Y/                  （详见 5.7）     │
│    ↓                                                        │
│  改路由表里 Accessor 对应的 key（X → Y）                    │
│  Accessor / UUID 不变，Lease 全部撤销，Policy 不自动改      │
└────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────┐
│  vault secrets disable X/                                  │
│    ↓                                                        │
│  清空该 UUID 子树（不可逆！） + 撤销路由表条目              │
└────────────────────────────────────────────────────────────┘
```
